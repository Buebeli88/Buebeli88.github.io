Was ist das Backend?

Frontend und Backend

Verschiedene Tools und Frameworks können zur Erstellung einer Webseite verwendet werden. Im Kern besteht das Frontend jedoch aus JavaScript, CSS, HTML und weiteren statischen Ressourcen wie Bildern oder Videos. Statische Dateien sind solche, die sich nicht verändern. Beim Aufruf einer Webseite werden diese Dateien an den Browser gesendet.

Das Aufrufen einer einfachen Webseite ist vergleichbar mit dem Bestellen von essen: Eine Bestellung wird aufgegeben, und sobald das Essen geliefert ist, befindet es sich vollständig im Besitz des Kunden. In diesem Vergleich entspricht das Frontend allem, was geliefert wird: Behälter, Besteck und das Essen selbst.

Frontend-Entwicklung wird oft als Client-seitige Entwickliung bezeichnet. Der "Client" ist dabei nicht zwingend der menschliche Beesucher, sondern der technische Anfrager - oft ein Browser, manchmal aber auch andere Anwendungen, mobile Geräte oder smart Geräte.

Das Backend umfasst alle Prozesse im Hintergrund, die eine Webseite funktionsfähig machen und Inhalte an Clients liefern.

Der Webserver

Woher kommt die Information, die an den Client geschickt wird? Die Antwort: vom Webserver.

"Server" kann in der informatik vieles bedeuten, hier geht es um den Webserver. Ein Webserver ist ein Prozess auf einem Computer, der Anfragen aus dem internet entgegennimmt und Antorten zurücksendet.

Jedes Mal, wenn eine Webseite aufgerufen wird, sendet der Broswer eine Anfrage an den Webserver der Website. jede Website hat mindestens einen Webserver. Grosse Unternehmen wie Facebook betreiben Tausende leistungsstarke Computer weltweit, die Webserver ausführen. Auch ein einfacher Webserver kann lokal betrieben werden.

Das Format der Anfrage und Antworten wird als Protokoll bezeichnet. Für Webseiten ist HTTP (Hypertext Transfer Protocol) das gängigste Protokoll. Beim Webseitenaufruf wird eine HTTP-Anfrage gesendet, um die notwendige Ressourcen zu erhalten.

Bei einfachen Webseiten reicht eine einzige Anfrage, auf die der Webserver mit allen notwendigen Inhalten antwortet. Dies nennt sich statische Website. "Statisch" bedeutet hier, dass die Inhalte sich nach dem Laden nicht mehr ändern. Solche Seiten eignen sich gut für einfache Inhalte wie persönliche Steckbriefe oder Fotoalben. Plattformen wie Twitter hingegen benötigen dynamische Inhalte - eine statische Website wäre hier ungegeignet.

Eine statische Website ist vergleichbar mit Lieferessen. Moderne Webanwendungen ähneln eher einem Restaurantbesuch mit Service: Bestellung nach Wunsch, Fragen an das Personal, individuelle Anpassungen - entsprechend komplex muss das Back-End sein.

Was ist das Back-End?

Beim Besuch von google.com wird die URL angefordert, aber nicht der Dateiname des Google-Doodle. Die Logik zur Auswahl der richtigen Inhalte liegt im Back-End. Moderne Anwendungen senden zudem nicht dieselben Dateien an alle Nutzer, sondern erzeugen dynamische Inhalte.

Vergleichbar mit einem Restaurantbesuch, bei dem das Essen individuell zusammengestellt wird, liefert das Back-End für jeden Besucher unterschiedliche, personalisierte Inhalte. zusätzlich laufen im Hintergrund viele Prozesse: Zutatenbeschaffung, Menüplanung, Personalverwaltung - analog dazu steuert das Back-End Geschäftslogik, Datenverarbeitung, Sicherheit und vieles mehr.

Diese Programmierlogik zur Bereitstellung dynamischer Inhalte, Sicherheitsmanagement, Zahlungsabwicklung uns. wird of als Application Server bezeichnet. Dieser kann Aufgaben wie Bestellbestätigungen oder die Ausführung komplexer Suchalgorithmen übernehmen.

Datenspeicherung

Daten spielen eine zentrale Rolle. Ein Grossteil der weltweit gespeicherten Daten wurden erst in den letzten Jahren erzeugt. online-Shops speichern Kreditkartendate, Streaming-Dienste merken sich Zeitstempel - all diese Daten müssen strukturiert gespeichert werden.

Moderne Back-Ends verwenden dafür Datenbanken - oft mehrere. Diese lassen sich grob in relationale und nicht-relationale (NoSQL) Datenbanken unterteilen. Realtionale Datenbanken speichern Informationen tabellarisch, NoSQL-Datenbanken arbeiten z.B. mit Schlüssel-Wert-Paaren oder Dokumentenmodellen.

SQL (Structured Query Language) wird zur Abfrage und Änderung relationaler Datenbaken verwendet. Bekannte relationale Datenbaken: MySKL, PostgreSQL. Bekannte NoSQL-Datenbanken: MongoDB, Redis.

Zusätzlich zur Datenbank benötigt das Back-End eine programmierbare Schnittstelle zur Bearbeitung und Analyse der Daten.

Was ist eine API

Beim Aurfurgen eines Artikels in einem online-Shop stammt der Preis aus der Datenbank. Bei Kauf muss der Lagerbestand aktualisiert werden. Viele dieser Prozesse im Back-End bestehen aus dem Lesen, Ändern oder Löschen von Daten.

Zur standardisierten Datenverarbeitung wird eine Web-API eingesetzt. API steht für Application Programming Interface und definiert Regeln zur Interatkion mit Anwendungsdaten - meist über HTTP-Anfragen.

Im Gegensatz zur klassischen HTTP-Anfrage zum Seitenaufruf sendet die API-Anfrage zusätzlich Informationen zur gewünschten Aktion: Daten erstellen, lese, aktualisieren oder löschen. Die Antwort enthält dann entsprechende Daten.

Beispiel: Bei Bestellbestätigung auf einer Website wird zusätzlich zur sichtbaren Ansicht eine unsichtbare Anfrage an die API gesendet, um die Datenbank zu aktualisieren.

Einige Web-APIs sind öffentlich zugänglich (z.B. Instagram), andere ausschliesslich für interne Zwecke.

Authentifizierung und Autorisierung

Zwei wiechtige Konzepte für die Serverseite: Authentifizierung und Autorisierung.

Authentifizierung prüft die Identität eines Benutzers - z.B. über Benutzername und Passwort. Diese Daten werden sicher in der Datenbank gespeichert und bei jedem Zugriff überprüft. Alternativ kann Authentifizierung über Drittanbieter wie Facebook, Goodle oder GitHub erfolgen.

Autorisierung steuert, wer auf welche inhalte und Funktionen zugreifen darf. Bearbeitungsseiten oder Löschfunktionen sind in der Regel nur bestimmten Nutzern zugänglich.

Ein sicheres und dynamisches Back-End integriert beide Prozesse, um personalisierte Inhalte sicher bereitzustellen.

Verschiedene Back-End-Stacks

Im Gegensatz zum Front-End (HTML, CSS, JavaScript) bietet das Back-End viele Möglichkeiten bei der Wahl von Programmiersprachen und Tools: PHP, Java, JavaScript; Python usw.

Zur Vereinfachung der Entwicklung werden meist Frameworks verwendet. Diese liefern vorgefertigte Strukturen und Funktionen.

Die Kombination von Technolien für Front-End und Back-End wird als Stack bezeischnet. Entwickler, die in beiden Bereichen arbeiten, sind Full-Stack-Entwickler.

Beispiele:

  • MEAN-Stack: MongoDB, Express.js, Angular, Nodesjs
  • LAMP-Stack: Linux, Apache, MySQL, PHP

Zusammenfassung

Das Back-End einer Webanwendung besteht aus vielen Komponenten:

  • Front-End: HTML, CSS, JavaScript und statische Dateien werden an den Client gesendet.
  • Webserver: Empfängt Anfragen, sendet Antworten.
  • Datenmanagement: Speicherung und Bearbeitung in relationalen oder NoSQL-Datenbanken.
  • Application Server: Führt Geschäftslogik aus, steuert Authentifizierung und Autorisierung.
  • Web-API: Schnittstelle für strukturierte Kommunikation mit Anwendungsdaten.
  • Stack: Gesamtheit aller verwendeten Technologien
