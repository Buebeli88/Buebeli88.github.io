JSON

Einleitung

In einer datenüberfluteten Welt wird der Umgang mit verschiedenen Datenformaten immer Wichtiger. Datenstrukturen müssen in ein Format übertragen werden,

das von anderen Programmiersprachen und Plattformen erkannt und gelesen werden kann. Glücklicherweise existiert ein solches Format zum Datenaustausch.

Was ist JSON

JSON (JavaScript Object Notation) ist ein beliebtes, sprachunabhängiges Standardformat zur Speicherung und zum Austausch von Daten. Es wurde von ECMA International übernommen - einer 1961 gegründeten Industrievereinigung zur Standardisierung von Informations- und Kommunikationssystemen.

JSON hat sich als De-facto-Standard etabliert, der die Speicherung und Übertragung von Daten zweischen allen Programmiersprachen erleichtert.

Typische Anwendungsfälle von JSON

JSON wird häufiger verwendet, um den Datentransfer in Webanwendungen zwischen einem Client (z.B. einem Webbrowser) und einem Server zu ermöglichen.

Beispiel: Beim Ausfüllen eines Webformulars wird das Formular von HTML in JavaScript-Objekte umgewandelt, anschliessend in JSON-Objekte konvertiert und zur Verarbeitung an einem Webserver gesendet. Solche Transaktionen reichen von einfachen Suchanfragen bis hin zu mehrseitigen Bewerbungsformularen.

Wenn Unternehmen ihre Daten öffentlich für andere Anwendungen bereitstellen - etwa Spotify seine Musikbibliothek oder Google Kartendaten - erfolgt die Formatierung dieser Informationen meist in JSON. So kann jede Anwendung, unabhängig von der verwendeten Sprache, die Daten empfangen und verarbeiten.

Beispiele beliebter Web-APIs, die JSON verwenden:

  • Google Maps
  • Google Auth 2.0 Authentication
  • Spotify Music Web API
  • LinkedIn Profile API

JSON-Syntax

Da Json aus der Programmiersprache JavaScript abgeleitet wurde, ähnelt es in der Syntax stark JavaScript-Objekten.

Beispiel: 

{
  "student": {
    "name": "Rumaisa Mahoney",
    "age": 30,
    "fullTime": true,
    "languages": ["JavaScript", "HTML", "CSS"],
    "GPA": 3.9,
    "favoriteSubject": null
  }
}

Wichtige Syntaxregeln:

  • Geschweifte Klammern {...} enthalten Objekte
  • Eckige Klammern [...] enthalten Arrays
  • Daten werden in Name-Wert-Parren gespeichert, getrennt durch einen Doppelpunkt :
  • Name-Wert-Paare sowie Array-Elemente werden durch ein Komman , getrennt
  • Keine abschliessenden Kommata erlaubt
  • Property-Name müssen in doppelten Anführungszeichen stehen (anders als in JavaScript)

JSON-Datentypen

Erlaubte Datentypen in JSON:

  • string (Zeichenkette in doppelten Anführungszeichen)
  • number (Ganzzahlen oder Fliesskommazahlen)
  • object (Name-Wert-Paare)
  • array (durch Kommas getrennte Liste von Werten)
  • boolean (true oder false)
  • null

In folgendem JSON-Beispiel sind alle Datentypen enthalten:

{
  "student": {
    "name": "Rumaisa Mahoney",
    "age": 30,
    "fullTime": true,
    "languages": ["JavaScript", "HTML", "CSS"],
    "GPA": 3.9,
    "favoriteSubject": null
  }
}

Nicht unterstützte Datentypen

Einige Datentypen werden in JSON nicht direkt unterstützt, z.B. Datumausgaben. Diese können jedoch als String gespeichert und in der jeweiligen Sprache weiterverarbeitet werden.

Beispiel für ein ISO 8691-Datum (gültiges Format): 

"2014-01-01T23:28:56.782Z"

Dieses Format enthält Informationen über Datum und Uhrzeit. Als String ist es schwer direkt nutzbar, doch moderne Programmiersprachen bieten Funktionen zur Konvertierung in lesbare und bearbeitbare Datumsobjekte.

Beispiel für eine konvertierte Darstellung:

Wed Jan 01 2014 13:28:56 GMT-1000 (Hawaiian Standard Time)

Zusammenfassung

JSON ist ein kompaktes, leicht lesbares Format zum Austausch strukturierter Daten zwischen Systemen.
