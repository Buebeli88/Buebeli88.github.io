Conditional Statements

Was sind bedingte Anweisungen (Conditional Statements)?

Im Alltag basieren viele Handlungen auf bestimmten Bedingungen. Ein typische Beispiel: Bei Müdigkeit erfolgt der Gang ins Bett, andernfalls beginnt der Tag mit deiner Aktivitäg.

Solche Entscheidungsprozesse lassen sich in Programmen mithilfe von bedingten Anweisungen (Conditional Statements) abbilden.

Eine bedingte Anweisung überprüft, ob eine bestimmte Bedingung erfüllt ist, und führt daraufhin eine Aktion aus. Wird die Bedingung nicht erfüllt, kann alternativ eine andere Anweisung ausgeführt werden.

Diese Form der Logik ermöglicht es Programmen, flexibel und situationsabhängig zu reagieren.

if - Anweisung

Viele Handlungen erfolgen auf Basis bestimmter Bedingungen, beispielsweise:

  • Bei schönem Wetter erfolgt ein Aufenthalt im Freien.
  • Läutet der Wecker, wird er ausgestellt.
  • Bei Müdigkeit erfolgt der Gang ins Bett.

Ähnliche Entscheidungsstruckturen lassen sich in der Programmierung mit einer if - Anweisung abbilden.

Beispiel:

if (true) {
  console.log('This message will print!');
}

Ausgabe:

This message will print!

Eine if - Anweisung besteht aus folgenden Elementen:

  1. Dem Schlüsselwort if, gefolgt von runden Klammern ()
  2. Innerhalb der Klammern befindet sich eine Bedingung, die entweder true (wahr) oder false (falsch) ergibt
  3. Anschliessend folgt ein Codeblock in geschweiften Klammern {}

Ergibt die Bedingung true, wird der Code im Block ausgeführt. Ist die Bedingung false, wird der Codeblock übersprungen.

if...else - Anweisung

Eine einfache if - Anweisung prüft eine Bedingung und führt den Codeblock nur aus, wenn diese Bedingung true ist.

Sollte die Bedingung jedoch false sein und eine alternative Aktion gewünscht wird, kommt die else - Anweisung zum Einsatz. Sie definiert einen Codeblock, der ausgeführt wird, wenn die Bedingung der if - Anweisung nicht erfüllt ist.

Beispiel:

if (false) {
  console.log('The code in this block will not run.');
} else {
  console.log('But the code in this block will!');
}

Ausgabe:

But the code in this block will!

Eine else - Anweisung steht immer in Verbindung mit einer if - Anweisung, zusammen bilden sie eine if...else - Anweisung.

Charakteristika einer if...else - Anweisung:

  • Die else - Anweisung folgt unmittelbar nach dem Codeblock der if - Anweisung.
  • Sie beginnt mit dem Schlüsselwort else.
  • Der Codeblock der else - Anweisung wird in geschweiften Klammern {} eingeschlossen.
  • Der Code innerhalb des else - Blocks wird nur ausgeführt, wenn die Bedingung der if - Anweisung den Wert false ergibt.

Vergleichsoperatoren

Beim Formulieren von Bedingungen, beispielsweise in if - Anweisungen, ist ein Vergleich von Werten oft erforderlich. Dazu dienen die sogenannten Vergleichsoperatoren (comparison operators).

Übersicht nützlicher Vergleichsoperatoen in JavaScript:

Operator / Bedeutung / Beispiel
< / Kleiner als / 3 < 5 => true
> / Grösser als / 10 > 7 => true
<= / Kleiner oder gleich / 4 <= 4 => true
>= / Grösser oder gleich / 6 >= 9 => false
=== / Gleich (Wert und Typ) / 5 === 5 => true
!== / Ungleich (Wert oder Typ) / 3 !== '3' => true

Wichtig:
Diese Operatoren vergleichen den Wert links mit dem Wert rechts und liefern stets ein Boolean - Ergebnis: true oder false.

Vergleichsausdrücke lassen sich als Fragen formulieren:

  • Antwort "Ja" => Ausdruck ergibt true
  • Antwort "Nein" => Ausdruck ergibt false

Grundaufbau von Vergleichsausdrücken:

  • Zwei Werte, die verglichen werden (z.B. Zahlen, Strings, Variablen).
  • Ein Vergleichsoperator, der die beiden Werte verbindet (z.B. >, <, >=, <=, ===, !==).

Logische Operatoren (Logical Operators)

Beim Arbeiten mit Bedingungen spielen Booleans - also die Werte true und false - eine zentrale Rolle. Um mehrere Bedingungen zu verknüpfen oder ihre Bedeutung zu verändern, stellt JavaScript drei logische Operatoren zur Verfügung.

Operator / Name / Beschreibung
&& / Und - Operator (AND) / Gibt true zurück, wenn alle Bedingungen erfüllt sind.
|| / ODER - Operator (OR) / Gibt true zurück, wenn eine Bedingung erfüllt ist.
! / NICHT - Operator (NOT / "Bang") / Kehrt den Warheitswert um (true zu false, false zu true).

Grundregeln:

  • && => Alle verknüpften Bedingungen müssen den Wert true haben
  • || => Es reicht aus, wenn eine der Bedingungen true ist
  • ! => Der Wahrheitswert wird umgekehrt

Anwendungsbeispiel: && (UND - Operator)

if (stopLight === 'green' && pedestrians === 0) {
  console.log('Go!');
} else {
  console.log('Stop');
}

In diesem Beispiel wird die Anweisung nur ausgeführt, wenn beide Bedingungen erfüllt sind. Ist auch nur eine davon false, wird der alternative Codeblock ausgeführt.

Anwendungsbeispiel: || (ODER - Operator)

if (day === 'Saturday' || day === 'Sunday') {
  console.log('Enjoy the weekend!');
} else {
  console.log('Do some work.');
}

Beim ODER - Operator genügt es, wenn eine der Bedingungen den Wert true hat, um den ersten Codeblock auszufürhen.

Anwendungsbeispiel: ! (NICHT - Operator)

let exited = true;
console.log(!exited);

let sleepy = false;
console.log(!sleepy);

Ausgabe:

false

true

Der NOT - Operator kehrt den Wahrheitswert eines Ausdrucks um. Wird true übergeben, ergibt sich false und umgekehrt.

Logische Operatoren erweitern die Ausdruckskraft von Bedingungen und ermöglichen differenzierte Entscheidungslogik in Programmen.

Truthy und Falsy

Nicht alle Bedingungen in Programmen basieren auf expliziten Boolean - Werten (true oder false). Auch andere Datentypen - wie Zeichenketten oder Zahlen - können in bedingten Ausdrücken verwendet und entsprechend ausgewertet werden.

Ein häufiger Anwendungsfall besteht darin, lediglich zu prüfen, ob eine Variable ein Wert zugewiesen wurde, unabhängig vom konkreten Inhalt.

Beispiel:

let myVariable = 'I Exist!';

if (myVariable) {
  console.log(myVariable);
} else {
console.log('The variable does not exist.');
}

In diesem Fall wird der Code im if - Block ausgeführt, da der Wert von myVariable als truthy gild. Obwohl es sich nicht um true im engerem Sinne handelt, wird der Wert in einem logischen Kontext als wahr interpretiert.

Was gild als falsy?
Ein falsy - Wert wird in einem booleschen Ausdruck als false interpretiert. Zu den falsy - Werten in JavaScript zählen:

  • 0
  • Leere Zeichenketten ("" oder '')
  • null (kein Wert vorhanden)
  • undefined (Variable ist deklariert, aber nicht initialisiert)
  • NaN (kein gültiger numerischer Wert)

Beispiel: Falsy - Wert in einer Bedingung

let numberOfApples = 0;

if (numberOfApples) {
  ('Let us eat apples!');
} else {
  console.log('No apples left!');
}

Ausgabe:

No apples left!

Die Unterscheidung zwischen truthy und falsy ist essenziell für das Verständnis bedingter Anweisungen in JavaScript. Sie ermöglicht eine flexible Handhabung von Variablen in logischen Ausdrücken - auch ohne explizite Booleans.

Zuweisung mit Truthy und Falsy

Die Konzepte truthy und falsy ermöglichen elegante Kurzschreibweisen in JavaScript - besonders bei der Zuweisung von Standardwerten.

Ein häufiger Anwendungsfall: Eine Variable, beispielsweise username, enthält unter Umständen kein Wert. Um dennoch einen brauchbaren Fallback bereitzustellen, kann eine bedingte Zuweisung erfolgen.

Klassische Schreibweise:

let username = '';
let defaultName;

if (username) {
  defaultName = username;
} else {
  defaultName = 'Stranger';
}

console.log(defaultName);

Ausgabe:

Stranger

In diesem Beispiel wird überprütf, ob username einen truthy Wert enthält. Ist dies nicht der Fall, erhält defaultName den alternativen Wert 'Stranger'.

Kurzschreibweise mit || (ODER - Operator)

let username = '';
let derfaultName = username || 'Stranger';

console.log(defaultName);

Ausgabe:

Stranger

Diese Schreibweise nutzt das Prinzip der Short - Circuit Evaluation:
Zuerst wird der linke Ausdruck ausgewertet. ist dieser truthy, wird er übernommen. Andernfalls greift der rechte Ausdruck als Ersatzwert.

Short - Circuit Evaluation mit dem || - Operator bietet eine effiziente Möglichkeit, Stantardwerte zuzuweisen - besonders dann , wenn Variablen möglicherweise lerr, undefiniert oder andersweitig falsy sind.

Ternärer Operator

Der ternäre Operator bietet eine kompakte Alternative zur klassischen if...else - Anweisung und ermöglich eine Kurzschreibweise bei bedingten Ausdrücken.

Beispiel mit if...else:

let isNightTime = true;

if (isNightTime) {
  console.log('Turn on the lights!');
} else {
  console.log('Turn off the lights!');
}

Dieselbe Logik mit dem ternären Operator:

isNightTime ! console.log('Turn on the lights!')
: console.log('Turn off the lights!');

Erläuterung

  • Die Bedingungen stehen vor dem Fragezeichen ?.
  • Danach folgen zwei Ausdrücke, getrennt durch einen Doppelpunkt:.
  • Ist die Bedinung wahr (true), wird der erste Ausdruck ausgeführt.
  • Ist die Bedingung (false), erfolgt die Ausführung des zweiten Ausdrucks.

Der ternäre Operator eignet sich für alle Bedingungen, die zu einem Wahrheitswert ausgewertet werden.

Der ternäre Operator ermöglicht eine übersichtliche prägnante Formulierung von Bedingungen mit zwei möglichen Ausgaben - eine elegante Alternative zur traditionellen if...else - Struktur.

else - if - Anweisung

Die else if - Anweisung erweitert die einfache if...else - Struktur um zusätzliche Bedingungen und ermöglicht somit mehr als zwei mögliche Ausgänge. Beliebig viele else if - Zweige lassen sich anfügen, um komplexe Entscheidungslogiken abzubilden.

Aufbau und Reihenfolge
Eine else if - Anweisung folgt stehts auf eine if - Anweisung und steht vor einer optionalen abschliessenden else - Anweisung. Sie nimmt eine Bedingung entgegen, die überprüft wird, falls alle vorherigen Bedingungen als falsch ausgewertet wurden.

Beispiel:

let stopLight = 'yellow';

if (stopLight === 'red') {   console.log('Stop!');
} else if (stopLight === 'yellow') {
  console.log('Slow down.');
} else if (stopLight === 'green') {
  console.log('go');
} else {
  console.log('Caution, unknown!');
}

Die Bedingungen werden der Reihe nach von oben nach unten grprüft. Sobald eine Bedingung als wahr (true) erkannt wird, wird der zugehörige Codeblock ausgeführt, und alle weiteren Prüfungen entfallen.

Im Beispiel ist die erste Bedingung stopLight === 'red' falsch, die zweite Bedingung stopLight === 'yellow' jedoch wahr. Deshalb erfolgt die Ausgabe "Slow down."; nachfolgende Bedingungen bleiben unerheblich. Sollte keine Bedingung zutreffen, greift der Code im else - Block.

if...else if...else - Ketten erlauben die Abbildung von mehrstufigen Entscheidungsprozessen, bei denen verschiedene Bedingungen sequenziell geprüft und passende Aktionen ausgeführt werden.

Das Schlüsselwort switch

else if - Anweisungen sind nützlich, wenn mehrere Bedingugen überprüft werden müssen. Häufig besteht jedoch die Aufgabe darin, einen Wert mit vielen möglichen Varianten zu vergleichen und jeden Fall unterschiedlich zu behandeln.

Beispiel mit else if:

let groceryItem = 'papaya';

if (groceryItem === 'tomato') {
  console.log('Tomatoes are $0.40');
} else if (groceryItem === 'papaya') {
  console.log('Papayas are $1.29');
} else {
  console.log('Invalid Item');
}

Dieser Code funktioniert, doch bei einer Vielzahl von Werten (zum Beispiel 100) wird der Code unübersichtlich und aufwendig.

Alternative: Die switch - Anweisung
Die switch - Anweisung bietet eine übersichtlichere und besser lesbare Möglichkeit, mehrere Fälle zu prüfen:

let groceryItem = 'papaya';

switch (groceryTiem) {
  case 'tomato' :
    console.log('Tomatoes are $0.49');
    break;
∇ case 'lime':
    console.log('Limes are $1.49');
    break;
  case 'papaya':
    console.log('Papayas are $1.29');
  ∇ break;
  default:
    console.log('Invalid item');
  ∇ break;
}

Ausgabe:

Papayas are $1.29

Erklärung der switch - Struktur:

  • Das Schlüsselwort switch wird gefolgt von runden Klammern (...), in denen ein Ausdruck oder Wert steht, der mit den einzelnen case - Werten verglichen wird. Hier ist das der Wert von groceryItem.
  • Innerhalb der geschweiften Klammern {...} folgen mehrere case - Blöcke
  • Jeder case prüft, ob der Wert des switch - Ausdrucks mit dem nachfolgenden Wert übereinstimmt.
  • Wird eine Übereinstimmung gefunden, wird der zugehörige Code ausgeführt.
  • Das Schlüsselwort break sorgt dafür, dass nach Ausführung des passenden Codes die switch - Anweisung verlassen wird. Ohne break würde die Ausführung in alle folgenden case - Blöcken "hineinfallen" (Fallthrough), was meist unerwünscht ist.
  • Die default - Anweisung am Ende fungiert als "Fangnetz" für alle Fälle, die keinem case entsprechen.

Die switch - Anweisung ist besonders geeignet, wenn viele mögliche Werte einzeln behandelt werden müssen. Sie schafft Klarheit und reduziert komplexe if...else if - Ketten.
