Variablen

In der Programmierung sind Variablen wie kleine Behälter für Informationen. Man kann sie sich als beschriftete Kästchen im Speicher eines Computers vorstellen – in ihnen liegt ein bestimmter Wert, etwa ein Benutzername, eine Kontonummer oder eine persönliche Begrüßung. Diese Informationen können später wieder abgerufen und genutzt werden.

Das Besondere an Variablen: Sie erlauben es uns, Daten mit verständlichen, aussagekräftigen Namen zu versehen. So bleibt der Programmcode nicht nur funktional, sondern auch nachvollziehbar – für uns selbst und für andere.

Zusammengefasst:
Variablen speichern Werte im Speicher und geben ihnen eine beschreibende Bezeichnung. Dabei können wir im Wesentlichen drei Dinge mit Variablen tun:

  • Eine Variable mit einem passenden Namen definieren
  • Einen Wert in dieser Variable speichern oder ihn bei Bedarf aktualisieren
  • Den gespeicherten Wert wieder abrufen

Wichtig ist: Eine Variable ist nicht der Wert selbst, sondern enthält ihn. Sie steht stellvertretend für den Wert und verweist durch ihren Namen darauf.

Stell dir das so vor:
Eine Variable ist wie ein farbiges Kästchen mit einem Etikett. Das Etikett ist der Name der Variable, und im Inneren liegt der gespeicherte Wert.

Eine Variable erstellen

var, let und const – Variablen in JavaScript
Mit der Einführung von ES6 (ECMAScript 2015) hielt eine ganze Reihe neuer Funktionen Einzug in JavaScript. Eine der bedeutendsten Neuerungen war die Einführung zweier neuer Schlüsselwörter: let und const. Mit ihnen lassen sich – genau wie mit dem bisherigen var – Variablen deklarieren. Vor ES6 stand Entwicklern ausschließlich das Schlüsselwort var zur Verfügung, um Variablen zu erstellen.

Regeln für Variablennamen

Beim Benennen von Variablen solltest du einige grundlegende Regeln beachten:

  • Variablennamen dürfen nicht mit einer Zahl beginnen.
    Richtig: user1 – Falsch: 1user
  • Groß- und Kleinschreibung spielt eine Rolle.
    myName und myname sind zwei verschiedene Variablen. Es ist jedoch keine gute Praxis, ähnliche Namen mit nur unterschiedlicher Groß-/Kleinschreibung zu verwenden – das führt schnell zu Verwirrung.
  • Vermeide reservierte Schlüsselwörter.
    Du darfst keine Wörter als Variablennamen verwenden, die bereits für die Sprache selbst reserviert sind (wie function, return, const usw.). Eine vollständige Liste findest du in der MDN-Dokumentation zu Schlüsselwörtern.

Variable var

Vor ES6 stand Entwicklern ausschließlich das Schlüsselwort var zur Verfügung, um Variablen zu erstellen.

Beispiel:

var myName = 'Arya';
console.log(myName);

Ausgabe:

Arya

Werfen wir einen genaueren Blick auf das Beispiel:

  • var ist ein JavaScript-Schlüsselwort und steht für "variable". Es wird verwendet, um eine neue Variable zu deklarieren.
  • myName ist der Name der Variable. Die Schreibweise folgt dem sogenannten camelCase: Dabei beginnt das erste Wort klein, und jedes weitere Wort startet mit einem Großbuchstaben (z. B. firstName, userScore).
  • = ist der Zuweisungsoperator. Er weist der Variablen den angegebenen Wert zu.
  • 'Arya' ist der Wert, der in der Variablen gespeichert wird. Man sagt auch: Die Variable wurde mit diesem Wert initialisiert.
  • Mit console.log(myName) wird der Wert der Variable in der Konsole ausgegeben.
