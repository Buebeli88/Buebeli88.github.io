Variablen

In der Programmierung sind Variablen wie kleine Behälter für Informationen. Man kann sie sich als beschriftete Kästchen im Speicher eines Computers vorstellen – in ihnen liegt ein bestimmter Wert,

etwa ein Benutzername, eine Kontonummer oder eine persönliche Begrüßung. Diese Informationen können später wieder abgerufen und genutzt werden.

Das Besondere an Variablen: Sie erlauben es uns, Daten mit verständlichen, aussagekräftigen Namen zu versehen. So bleibt der Programmcode nicht nur funktional,

sondern auch nachvollziehbar – für uns selbst und für andere.

Zusammengefasst:
Variablen speichern Werte im Speicher und geben ihnen eine beschreibende Bezeichnung. Dabei können wir im Wesentlichen drei Dinge mit Variablen tun:

  • Eine Variable mit einem passenden Namen definieren
  • Einen Wert in dieser Variable speichern oder ihn bei Bedarf aktualisieren
  • Den gespeicherten Wert wieder abrufen

Wichtig ist:
Eine Variable ist nicht der Wert selbst, sondern enthält ihn. Sie steht stellvertretend für den Wert und verweist durch ihren Namen darauf.

Stell dir das so vor:
Eine Variable ist wie ein farbiges Kästchen mit einem Etikett. Das Etikett ist der Name der Variable, und im Inneren liegt der gespeicherte Wert.

Eine Variable erstellen

var, let und const – Variablen in JavaScript

Mit der Einführung von ES6 (ECMAScript 2015) hielt eine ganze Reihe neuer Funktionen Einzug in JavaScript. Eine der bedeutendsten Neuerungen war die Einführung zweier neuer Schlüsselwörter: let und const.

Mit ihnen lassen sich – genau wie mit dem bisherigen var – Variablen deklarieren. Vor ES6 stand Entwicklern ausschließlich das Schlüsselwort var zur Verfügung, um Variablen zu erstellen.

Wann sollte let oder const verwendet werden?

  • Wenn sich der Wert der Variable im Laufe des Programms ändern soll, verwende let.
  • Wenn der Wert fest bleiben soll, ist const die richtige Wahl.

Regeln für Variablennamen

Beim Benennen von Variablen solltest du einige grundlegende Regeln beachten:

  • Variablennamen dürfen nicht mit einer Zahl beginnen.
    Richtig: user1 – Falsch: 1user
  • Groß- und Kleinschreibung spielt eine Rolle.
    myName und myname sind zwei verschiedene Variablen. Es ist jedoch keine gute Praxis, ähnliche Namen mit nur unterschiedlicher Groß-/Kleinschreibung zu verwenden – das führt schnell zu Verwirrung.
  • Vermeide reservierte Schlüsselwörter.
    Du darfst keine Wörter als Variablennamen verwenden, die bereits für die Sprache selbst reserviert sind (wie function, return, const usw.). Eine vollständige Liste findest du in der MDN-Dokumentation zu Schlüsselwörtern.

Variable: var

Vor ES6 stand Entwicklern ausschließlich das Schlüsselwort var zur Verfügung, um Variablen zu erstellen.

Beispiel: 

var myName = 'Arya';
console.log(myName);
// Ausgabe: Arya

Werfen wir einen genaueren Blick auf das Beispiel:

  • var ist ein JavaScript-Schlüsselwort und steht für "variable". Es wird verwendet, um eine neue Variable zu deklarieren.
  • myName ist der Name der Variable. Die Schreibweise folgt dem sogenannten camelCase: Dabei beginnt das erste Wort klein, und jedes weitere Wort startet mit einem Großbuchstaben (z. B. firstName, userScore).
  • = ist der Zuweisungsoperator. Er weist der Variablen den angegebenen Wert zu.
  • 'Arya' ist der Wert, der in der Variablen gespeichert wird. Man sagt auch: Die Variable wurde mit diesem Wert initialisiert.
  • Mit console.log(myName) wird der Wert der Variable in der Konsole ausgegeben.

Variable: let

Das Schlüsselwort let, wurde mit ES6 eingeführt. Es ermöglicht die Deklaration von Variablen, deren Werte später geändert (neu zugewiesen) werden können.

Beispiel: 

let meal = 'Enchiladas';
console.log(meal);
// Ausgabe: Enchiladas

meal = 'Burrito';
console.log(meal)
// Ausgabe: Burrito

Ein weiterer wichtiger Aspekt bei der Verwendung von let (wie auch bei var) ist, dass man eine Variable deklarieren kann, ohne ihr sofort einen Wert zuzuweisen.

In diesem Fall wird die Variable automatisch mit dem Wert undefined initialisiert: 


let price;
console.log(price);
// Ausgabe: undefinded

price = 350;
console.log(price);
// Ausgabe: 350

Wichtig zu beachten:

  • Wird einer mit let deklarierten Variable kein Wert zugewiesen, erhält sie automatisch den Wert undefined.
  • Der Variable kann später problemlos ein neuer Wert zugewiesen werden.

Mit let deklarierte Variablen sind also flexibel, da ihre Werte im Laufe des Programms verändert werden können.

Variable: const

Das Schlüsselwort const wurde mit ES6 eingeführt und steht für „Konstante“. Es wird verwendet, um Variablen zu deklarieren, deren Wert nicht verändert werden soll.

Ähnlich wie bei let und var kann auch mit const ein beliebiger Wert gespeichert werden. Die Syntax zur Deklaration ist ebenfalls identisch: 

const myName = 'Gilberto';
console.log(myName);
// Ausgabe: Gilberto

Der entscheidende Unterschied:

Eine mit const deklarierte Variable darf nicht neu zugewiesen werden. Versucht man es dennoch, führt das zu einem TypeError.

Ein weiterer wichtiger Punkt:

Konstanten müssen bei der Deklaration sofort einen Wert erhalten. Andernfalls wirft JavaScript einen SyntaxError.

Mathematische Zuweisungsoperatoren

Mit mathematischen Zuweisungsoperatoren lassen sich Werte von Variablen berechnen und direkt aktualisieren.

Beispiel: 

let w = 4;
w = w +1;

console.log(w);
// Ausgabe: 5

In diesem Beispiel erhält die Variable w zunächst den Wert 4. Anschließend wird 1 addiert und das Ergebnis erneut w zugewiesen. Das Resultat ist 5.

Vereinfachte Schreibweise
Die gleiche Operation kann durch den Zuweisungsoperator += abgekürzt werden: 

let w = 4;
w += 1;

console.log(w);
// Ausgabe: 5

Der Operator += addiert den angegebenen Wert zur Variable und weist das Ergebnis derselben Variable zu.

Weitere mathematische Zuweisungsoperatoren:

Operator Beschreibung Entspricht
+= Addition und Zuweisung x += 2 => x = x + 2
-= Subtraktion und Zuweisung x -= 5 => y = y - 5
*= Multiplikation und Zuweisung x *= 3 => x = x * 3
/= Division und Zuweisung x /= 2 => x = x / 2

Mathematische Zuweisungsoperatoren ermöglichen es, Rechenoperationen direkt mit einer Variablen zu verknüpfen.

Dies führt zu kürzerem, übersichtlicherem und effizienterem Code.

Inkrement - und Dekrement - Operator

Zu den mathematischen Zuweisungsoperatoren zählen auch der Inkrementoperator (++) und der Dekrementoperator (--).

Der Inkrementoperator erhöht den Wert einer Variablen um 1, während der Dekrementoperator den Wert um 1 verringert.

Beispie:

let a = 10;
a++;
console.log(a);
// Ausgabe: 11

let b =10;
b--;
console.log(b);
// Ausgabe: 19

Wie bei den Operatoren +=, -=, *=, /= wird auch hier der aktuelle Wert einer Variablen verändert und der neue Wert gespeichert.

Durch die Verwendung dieser Operatoren lassen sich einfache Erhöhungs- oder Verringerungsvorgänge effizient darstellen und umsetzen.

String - Verkettung mit Variablen

Strings können nicht nur direkt verwendet, sondern auch miteinander kombiniert werden. Dies erfolgt häufig durch die sogenannte String-Verkettung.

Der +-Operator dient dazu, mehrere String-Werte miteinander zu verbinden – unabhängig davon, ob es sich um direkte Zeichenfolgen oder um Variablen handelt, in denen diese gespeichert sind. 

let myPet = 'armadillo';
console.log('I own a pet' + myPet + '.');
// Ausgabe: I own a pet armadillo.

In diesem Beispiel wird der Variable myPet der Wert 'armadillo' zugewiesen. Anschließend erfolgt die Verkettung aus drei Bestandteilen: dem Text 'I own a pet ', dem in myPet gespeicherten String und dem abschließenden Punkt '.'.

Die String-Verkettung stellt eine grundlegende Methode dar, um dynamisch zusammengesetzte Textausgaben zu erzeugen – insbesondere im Zusammenspiel mit Variablen.

String - Interpolation

Seit der ES6-Version von JavaScript besteht die Möglichkeit, Variablen direkt in Zeichenketten einzufügen.

Dies geschieht mithilfe sogenannter Template Literals.

Beispiel:

const myPet = 'armadillo'; console.log(`I own a pet ${myPet}.`);
// Ausgabe: I own a pet armadillo.

Ein Template Literal wird mit sogenannten Backticks ` umschlossen.

Innerhalb eines solchen Literals können Platzhalter in der Form ${variable} verwendet werden. An diesen Stellen wird der Wert der jeweiligen Variable automatisch eingefügt.

Durch die Interpolation des Ausdrucks `I own a pet ${myPet}.` entsteht der String:

I own a pet armadillo.

Einer der grössten Vorteile von Template Literals liegt in der verbesserten Lesbarkeit des Codes. Die Struktur des resultierenden Text lässt sich auf einen Blick erfassen, ohne zusätzliche Verkettungen oder Escape - Zeichen für Anführungszeichen.

Durch den Einsatz von Template Literals wird die String - Erzeugung übersichtlicher und effizienter gestaltet besonders bei längeren oder dynamischen zusammgesetzten Texten.

Der typeof - Operator

In der Programmierung spielt die Kenntnis des jeweiligen Datentyps einer Variable eine wichtige Rolle. Zur Überprüfung des Datentyps kann der Operator typeof eingesetzt werden.

Der typeof - Operator analysiert den Wert, der sich rechts von ihm befindet, und gibt einen String zurück, der den jeweiligen Datentyp beschreibt.

Beispiel:

const unknown1 ? 'foo';
console.log(typeof unknown1);
// Ausgabe: string

const unknown2 = 10;
console.log(typeof unknown2);
// Ausgabe: number

const unknown3 = true;
console.log(typeof unknown3);
// Ausgabe: boolean

Im ersten Beispiel enthält die Variable unknown1 den Wert 'foo', also eine Zeichenkette. Die Anwendung von typeof auf diesem Wert liefert daher das Ergebnis 'string'.

Die Verwendung von typeof ist besonders hilfreich bei dynamisch typisierten Sprachen wie JavaScript, um den aktuellen Typ einer Variable jederzeit ermitteln zu können.
