HTML Struktur

In HTML bezieht sich die Struktur auf die Anordnung und Organisation von Elementen innerhalb eines Dokuments, um eine sinnvolle und übersichtliche Darstellung von Inhalten zu gewährleisten.

Diese Struktur wird durch HTML-Elemente wie Überschriften, Absätze, Listen und andere Tags definiert, die dem Browser mitteilen, wie der Inhalt angezeigt werden soll.

HTML hat eine Familienbaumstruktur. Ein Element kann Eltern, Grosseltern, Geschwister, Kinder, Enkelkinder, etc. haben.

Beispiel: 


<body>
  <div>
    <h1>It's div's child and body's grandchilde</h1>
    <h2>It's h1's sibling</h2>
  </div>
</body>

Indentation

Indentation in HTML bezieht sich auf das Hinzufügen von Leerzeichen oder Tabs am Anfang von Codezeilen, um die Struktur des HTML-Dokuments visuell zu verbessern.

Obwohl HTML-Code auch ohne Einrückung funktioniert, macht die richtige Indentation den Code lesbarer und einfacher zu verwalten, insbesondere bei komplexen Strukturen mit verschachtelten Elementen.

Comments

Ein HTML-Kommentar ist eine Textnotiz innerhalb eines HTML-Dokuments, die vom Browser ignoriert und nicht auf der Webseite angezeigt wird.

Er dient dazu, den Code für Entwickler verständlicher zu machen, Anmerkungen zu hinterlassen oder temporär Code auszublenden.

Beispiel:


<!-- Main Site Content -->
<div>Content</div>

oder


<!--

Comments can be 
multiple Lines long.
-->

Whitespace

In HTML bezieht sich Whitespace auf Leerzeichen, Tabulatoren, Zeilenumbrüche und andere Zeichen, die keine sichtbaren Inhalte darstellen, aber dennoch Speicherplatz belegen und zur Strukturierung des Codes dienen.

Browser interpretieren Whitespace in HTML oft als einzelne Leerzeichen, d.h. mehrere aufeinanderfolgende Whitespace-Zeichen werden zu einem einzigen zusammengefasst.

Beispiel:

Code:


<p>Test Paragraph</p>
<!-- Der "Whitespace" der über und unter dieser Zeile ist, wird
vom Browser ignoriert -->

<p>Another Test Paragraph.</p>
<!-- this will sitright under the first Paragraph, no extra space between. -->

HTML:

Test Paragraph
Another Test Paragraph.

Semantic

Semantik in HTML bezieht sich auf die Verwendung von HTML-Elementen, die ihre Bedeutung und Funktion im Inhalt einer Webseite klar definieren, anstatt sie nur als generische Container (wie <div> oder <span>) zu verwenden.

Es geht also darum, den Inhalt basierend auf seiner Bedeutung zu strukturieren, nicht nur basierend auf seinem Aussehen.

  • Statt <div class="navigation">, sollte <nav> verwendet werden, um eine Navigation zu kennzeichnen.
  • Statt <div class="footer">, sollte <footer> verwendet werden, um den Fußbereich der Seite zu kennzeichnen.
  • Statt <div class="article">, sollte <article> verwendet werden, um einen eigenständigen Inhalt wie einen Blogartikel zu kennzeichnen.
  • Statt <div class="main">, sollte <main> verwendet werden, um den Hauptinhalt einer Seite zu kennzeichnen.
