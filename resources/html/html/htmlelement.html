HTML Element

Ein HTML-Element ist ein Baustein eines HTML-Dokuments, der dem Browser Anweisungen gibt, wie ein bestimmter Inhalt dargestellt und interpretiert werden soll.

Es besteht in der Regel aus einem öffnenden Tag, einem Inhalt und einem schließenden Tag. Einige Elemente sind jedoch "selbstschließende" und benötigen kein End-Tag.

<html> Element

Das <html> Element sollte direkt nach dem <!DOCTYPE html> eingefügt werden

und beinhaltet den ganzen Inhalt und die Struktur des Dokuments.

Beispiel:

<!DOCTYPE html>
<html>
  Inhalt und Struktur der Seite.
</html>

<head> Element

Das <head> Element beinhaltet generelle Informationen über die HTML Seite, die nicht im Browser angezeigt werden.

Diese Informationen nennt man auch "Metadaten". Title oder Linkts zu Stylesheets werden hier üblicherweise angegeben.

Beispiel:

<!DOCTYPE html>
<html>
  <head>
    <!-- Metadaten werden hier eingefügt-->
  </head>
</html>

<meta/> Element

<title> Element

Das <title> Element zeigt den Titel der Seite im Browser/Tab an. Das Element wird nur im <head> Element eingefügt.

Beispiel:

<!DOCTYPE html>
<html>
  <head>
    <title>Titel der HTMLSeite</title>
  </html>
</html>

<body> Element

Im <body> Element ist der ganze Inhalt und die Struktur, der im Browser dargestellten Seite. Es kann nur ein <body> Element pro Dokument existieren.

Beispiel:

<body>
  <h1>Learn to code with CodeAcademy</h1>
</body>

<h1> bis <h6> Heading Elemente

Ein HTML Dokument kann 6 verschiedene Heading Elemente darstellen. Vom grössten <h1> bis zum kleinsten <h6>

Beispiel:

<h1>Breaking News</h1>
<h2>This is the 1st subheading</h2>
<h3>This is the 2nd subheading</h3>
...
<h6>This is the 5th subheading </h6>

<p> Paragraph Element

Dieses Element beinhaltet einen Textblock.

Beispiel: 

<p>This is a block of text! Lorem ipsum dolor sit amet, consectetur adipisicing elit.</p>

<br> Line Break Element

Ein <br> Line Break Element ist ein Zeilenumbruch, der vom Browser auch dargestellt wird. (Im Gegensatz zum «normalen» Zeilenumbruch, siehe "Whitespace")

Dieses Element nutzt nur einen "opening tag" und KEINEN "closing tag".

Beispiel:

Code:

<p>
  A line break haiku. <br>
  Poemsare a great use case. <br>
  Oh joy! A linebreak.
</p>

HTML:

A line break haiku.
Poems are a great use case.
Oh joy! A line break.

<strong> Strong Element

Das <strong> Element setzt den Text zwischen dem "opening tag" und "closing tag" auf bold (Fett). Dies kann zusätzlich zu CSS funktionen über HTML benutzt werden.

(Wird aber nur die "normale" bold funktion übernehmen, welche in CSS für 400 steht)

Beispiel:

Code:

<p>
  This is <strong>important</strong>text!
</p>

HTML:

This is important text!

<em> Emphasis Element

Dieses Element wird den Text zwischen dem "opening tag" und "closing tag" auf italicize setzten.

Dies kann zusätzlich zu CSS funktionen über HTML benutzt werden.

Beispiel:

Code:

<p>
  This <em>word</em> will be emphasized
  in italics.
</p>

HTML:

This word will be emphasized in italics.

<div> Div Element

Das <div> Element wird genutzt um "Boxen" (container) zu erstellen, welche das HTML-Dokument in Sektionen aufteilt. Ein <div> Element kann sogenannten "flow content" beinhalten. (z.B. headings, paragraphs, links, images, etc…)

Div steht für "division". In Zusammenhang mit CSS und andere Tools (z.B. JavaScript) werden hier auch oft noch class und ID benutzt, um die <div> Container zu benennen, resp. anzusteuern.

Beispiel:

<div>
  <h1>A section of grouped elements</h1>
  <p>Here's some text for the section</p>
</div>
<div>
  <h1>Second section of groupedelements</h1>
  <p>Here's some text</p>
</div>

<span> Span Element

Das <span> Element ist ein sogenannter "inline" Container für Text und kann zum Stylen dieser Gruppe benutzt werden. Jedoch ist das <span> Element ein allgemeiner Container um einen Teil des Textes,

eines grösseren Text zu trennen. Sollten andere Semantische Elemente zur Verfügung stehen sollten diese Vorrang haben.

Beispiel:

<p><span>This text</span> may be styleddifferently than the surrounding text.</p>

<li> List Item Element

Ein List Item (Listenelement), repräsentiert durch das HTML-Element <li>, ist ein einzelnes Element in einer Liste. Es wird verwendet, um Elemente in geordneten Listen (<ol>), ungeordneten Listen (<ul>) oder Menüs (<menu>) darzustellen.

In ungeordneten Listen und Menüs werden Listenelemente standardmäßig mit Aufzählungszeichen dargestellt, während sie in geordneten Listen mit einer Nummerierung (oder einem anderen Zähler) versehen werden.

<ol> Ordered List Element

Das <ol> Element generiert standartmässig eine nummerierte Liste.

Beispiel:

Code:

<ol>
  <li>Preheat oven to 325F</li>
  <li>Drop cookie dougth</li>
  <li>Bake for 15min</li>
</ol>

HTML:

  1. Preheat oven to 325F
  2. Drop cookie dougth
  3. Bake for 15min

<ul> Unordered List Element

Das <ul> Element generiert eine nicht nummerierte Liste. Standartmässig werden hier Punkte statt Nummern als Listenelement dargestellt.

Beispiel:

Code:

<ul>
  <li>Play more music</li>
  <li>Read more Books</li>
</ul>

HTML:

  • Play more music
  • Read more Books

<a> Anchor Element

Das <a> Element wird benutzt um im HTML-Dokument einen Hyperlink zu erstellen. Der Hyperlink kann zu einer anderen Website, einem File auf dem gleichen Server, ein "punk" auf der gleichen Seite oder einer anderen URL über das «href» Attribut weisen.

Das "href" Attribut bestimmt das Ziel des Hyperlinks. Text oder auf Bilder die zwischen dem "opening tag" und "closing tag" sind, werden zum Hyperlink.

Beispiel:

<a href="http://www.codeacademy.com">Visit this site</a>

oder

<a href="https://www.codeacademy.com">

  <img src="logo.jpg">Click this image
</a>

<target> Target Attribute

Das <target> Attribut bestimmt, wo der Hyperlink geöffnet wird. Der wird "_blank" wird den Link standartmässig in einem neuen «Tab» öffnen, in einem älteren Browser evtl.

auch in einem neuen "Fenster" oder die Einstellungen im Browser wurden so geändert das immer ein neues "Fenster" geöffnet wird.

Beispiel:

<a href="https://google.com" target="_blank">This anchor
element links to google and will open in a new tab or
window.</a>

Link to a different part of the page #

Das <a> Anchor Element kann Hyperlinks zu verschiedenen Teilen einer Seite führen.

Dazu wird im "href" Attribute ein "#" benutzt, gefolgt von der "ID" des gewünschten Elements.

Beispiel:

<div>
  <p id="id-of-element-to-link-to">A different part
  of the page!</p>
</div>
<a href="#id-of-element-to-link-to">Take me to a differentpart of the page</a>

File Path

Ein URL Weg in HTML kann ein "absoluter" Weg sein, wie z.B. eine komplette URL. (Beispiel: https://developer.mozilla.org/en-US/docs/Learn) Oder ein Relativer File Weg,

der zu einem Lokalem Dokument im gleichen Ordner oder auf dem gleichen Server führt. Z.B. .style.css. Ein Relativer Weg beginnt mit ./ gefolgt vom "file path" zur Lokalen Datei.

Beispiel:

<a href="https://developer.mozilla.org/en-US/docs/web">The URL
for this anchor element is an absolute file path.</a>

            oder

<a href=".about.html">The URL for this anchor element is a
relative file path.</a>

<img> Image Element

Ein <img> Element hinterlegt ein Bild im Dokument. Das "src" Attribut beinhaltet die URL des Bildes und ist demnach Pflicht.

Das <img> Element ist ein sogennantes "empty element", was bedeutet das es keinen "closing tag" besitzt.

Beispiel:

Code:

<img src="resources/images/cutecat.jpg" />

HTML:

"alt" Attribute

Im <img> Element sollte immer auch ein alternativer Text stehen, welcher über das "alt" Attribut erteilt wird. Dieser alternative Text wird nur dann, im Browser dargestellt, wenn das Bild nicht "geladen" werden kann. (Z.B. Das Bildformat wird vom Browser nicht erkannt, das Bild wurde blockiert, ein Fehler in der URL.)

Dieser Text wird auch von einen "Screen Reader" vorgelesen und hilft demnach den Inhalt der Seite auch von Sehbeeinträchtigten besser zu verstehen. Ein "alt" Attribut funktionier wie ein "ARIA-Label" hat aber noch zusätzliche Funktionen und sollte daher bei Bildern Vorrang haben.

Wenn ein "alt" Attribut verwendet wird, sollten folgene Sachen beachtet werden:

  1. Generell sollte der "value" vom "alt" Attribut nur das Bild Beschreiben.
  2. Für Bilder die als Hyperlink dienen, sollte das "alt" Attribut auch den Inhalt des Links beschreiben.
  3. Bei einem Bild ohne Informationen (wie ein dekorativer Rahmen) sollte der Inhalt des "alt" Attributs leer sein, das Attribut sollte aber nicht weggelassen werden.
  4. Sollte ein Bild schon beschrieben werden. (z.B. In einem Textabschnitt unter dem Bild selbst) Wird auch hier ein leeres "alt" Attribut verwendet. Die Beschreibung sollte nicht dupliziert werden durch das "alt" Attribut.
  5. Der Inhalt eines "alt" Attributs sollte nicht mehr als 150 Zeichen beinhalten.

Beispiel:

<img src="path/to/image" alt="text describing image" />

<video> Video Element

Ein <video> Element wird im Browser einen "Media Player" anzeigen. Im "src" Attribut wir die URL des Videos hinterlegt. Das hinzufügen des "control" Attribute, gibt dem Video noch die Wiedergabefunktionen wie "play" oder "stop".

Ein <video> Element benutzt im Gegensatz des "img" Elements einen "opening tag" und einen "closing tag". Der Inhalt dazwischen ist der "fallback" text (gleich wie ein "alt" Attribut beim "img" Element) und wird nur dargestellt, wenn das Video nicht funktioniert.

Seiten wie YouTube bieten auch Optionen wie "Einbetten/Embed" an mit ganzem Code.

Beispiel:

Code:

<video src="resources/videos/cutecats.mp4" control>
  Video not supportet
< /video>

HTML Attribute

Ein Attribut wird immer im "opening tag" eines Elements verwendet. Durch das hinzufügen eines Attributs kann ein Element spezifischer angepasst werden und auch für weitere Nutzungen anvisiert werden.

(Mit Hilfe von "id" und "class") Grundsätzlich sollte zum anvisieren eines Elements zum stylen über CSS ein "class" Attribut dem "id" Attribut bevorzugt werden. (ID's werden in anderen Tools mehr benutzt und sind einzigartig)

Beispiel:

Code:

<p class="my-paragraph" style="color: green;">
  Here's some text for a paragraph that is
  being altered by HTML Attributes
</p>

oder

<p id="my-paragraph" style="color: red;">
  Here's some text of a paragraph that is
  being altered by HTML Attributes
</p>

HTML:

Here's some text for a paragraph that is being altered by HTML Attributes

oder

Here's some text of a paragraph that is being altered by HTML Attributes

Unique ID Attributes

Im HTML-Dokument können Elemente mit spezifischen und einzigartigen "id" Attributen versehen werden, welche zur Unterscheidung anderer "gleichen" Elemente dient. Bei gebrauch, können "id" Attribute direkt von CSS oder JavaScript anvisiert werden und diese Elemente zu manipulieren, formatieren oder ihnen spezifische Instruktionen zu geben.

Der Inhalt des "id" Attributs sollte nur Buchstaben (a-Z), Nummern (0-9), hyphens (-), underscores (_) und periods(.) enthalten. Genauso werden keine Leerzeichen benutzt.

Beispiel:

<h1 id="A1">Hello World</h1>
